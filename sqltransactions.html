SQL Transactions

A transaction is a logical unit of work that consists of one or more SQL statements. Transactions are used to ensure the integrity of data in a database by guaranteeing that either all or none of a series of related SQL statements are executed.


Transactions are typically used in situations where multiple SQL statements need to be executed as a single unit of work, such as transferring funds between bank accounts or updating inventory levels in an e-commerce system.


In SQL, transactions can be initiated using the BEGIN TRANSACTION statement, and can be committed or rolled back using the COMMIT and ROLLBACK statements, respectively.


For example, the following SQL statements would initiate a transaction, insert a new row into the "orders" table, and then commit the transaction:


BEGIN TRANSACTION; INSERT INTO orders (order_id, customer_id, order_date) VALUES (1, 1001, '2023-05-10'); COMMIT;

If an error occurs during the transaction, the ROLLBACK statement can be used to undo any changes that were made to the database:


BEGIN TRANSACTION; INSERT INTO orders (order_id, customer_id, order_date) VALUES (1, 1001, '2023-05-10'); UPDATE inventory SET quantity = quantity - 1 WHERE product_id = 1234; IF @@ERROR <> 0 ROLLBACK; ELSE COMMIT;

The above SQL statements initiate a transaction, insert a new row into the "orders" table, and update the inventory level for a product. If an error occurs during the update statement, the ROLLBACK statement is executed to undo the changes made to the database.
